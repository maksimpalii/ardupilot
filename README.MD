# 🚁 Автопілот БПЛА: Навігація та компенсація вітру (ArduPilot SITL)

Цей репозиторій містить розв'язок тестового завдання з розробки скрипта керування квадрокоптером за допомогою `dronekit` у симуляторі `ArduPilot SITL`. 

**Головна мета:** Здійснити автономний зліт, переліт з Точки А в Точку Б (на висоті 100м) у ручному режимі **STABILIZE**, компенсувати заданий боковий вітер (3 м/с) та виконати прецизійну посадку в ціль за допомогою прямого керування ШІМ-каналами (RC Overrides).

## ✨ Особливості реалізації (Інженерні рішення)

Замість базового наведення по азимуту, яке в умовах вітру призводить до "ефекту унітазу" (кружляння над ціллю), у цьому скрипті реалізовано просунуту векторну математику та незалежне керування осями:

1. **Векторна навігація (Body-Frame):** Глобальні координати (GPS) перетворюються у локальні вектори похибки дрона (`Err_Fwd` - Вперед/Назад, `Err_Right` - Вліво/Вправо). Це дозволяє дрону "ковзати" в просторі, незалежно компенсуючи знесення вітром за допомогою крену (Roll) та тангажу (Pitch).
2. **Власні ПІД-регулятори:** Написано кастомний клас `PIDController` із захистом від накопичення помилки (Anti-Windup). Створено 4 незалежні контури керування для каналів: Roll, Pitch, Throttle та Yaw.
3. **Прецизійна посадка ("Віртуальна труба"):** Спуск активується лише тоді, коли дрон знаходиться ідеально над ціллю (відхилення < 1.5 м). Якщо порив вітру зміщує коптер, спуск призупиняється до відновлення позиції.
4. **Розширена Телеметрія:** Скрипт виводить форматовані дані в консоль у реальному часі та паралельно записує детальний лог (`flight_log_full.csv`) з частотою 10 Гц для аналізу роботи ПІД-регуляторів.

## 🛠 Технології
* Python 3
* `dronekit` (MAVLink комунікація)
* `math`, `csv`, `time` (стандартні бібліотеки)

## 📂 Структура коду
* `PIDController` — клас для обчислення керуючих сигналів (ШІМ) на основі похибки, з урахуванням пропорційної, інтегральної та диференціальної складових.
* `get_body_frame_errors()` — ключова функція, що відповідає за матрицю повороту та проектування GPS-координат на осі самого дрона.
* `main()` — головний цикл (Control Loop), що працює на частоті 10 Гц, розраховує похибки, застосовує ПІД-регулятори та відправляє `RC overrides`.
